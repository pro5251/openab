# 任務記錄 Skill

> **觸發條件**：每次完成一項任務後，或使用者要求記錄任務時。

## 用途

在 `tasks/` 目錄建立一份 Markdown 格式的任務記錄，方便追蹤歷史決策、方便其他 Agent 參考、也作為團隊知識庫。

---

## 執行步驟

### Step 1：確認任務資訊

整理以下資訊再開始撰寫：

- **需求議題**：任務的簡短標題（中文或英文均可，用於檔名）
- **日期時間**：執行當下的本地時間
- **需求來源**：誰指派（梅長蘇 / 使用者直接）
- **完成項目**：做了什麼
- **技術決策**：重要的設計或技術選擇（若有）

### Step 2：建立任務記錄檔案

**檔名格式**：`yyyy-MM-dd_HHmmss_[需求議題].md`

範例：`2026-04-29_175320_後端登入-API實作.md`

**存放路徑**：`/workspace/tasks/` 或專案根目錄下的 `tasks/`

> 若工作目錄不在專案根目錄，使用絕對路徑：`{working_dir}/tasks/`

### Step 3：撰寫內容

使用以下模板：

```markdown
# 任務：{需求議題}

**日期**：{yyyy-MM-dd HH:mm:ss}
**Agent**：{你的名字與角色}
**需求來源**：{指派者或觸發來源}

---

## 需求描述

{完整說明這個任務要解決什麼問題、達成什麼目標}

## 完成項目

- {項目 1}
- {項目 2}
- ...

## 技術決策

{重要的設計選擇、架構決策、技術取捨說明。若無重大決策，填「無」}

## 影響範圍

**新增檔案：**
- `{路徑}` — {說明}

**修改檔案：**
- `{路徑}` — {修改內容摘要}

## 備註

{其他補充說明、已知限制、後續建議事項。若無，填「無」}
```

### Step 4：確認寫入成功

建立後確認：

```bash
ls tasks/ | tail -5
```

---

## 注意事項

- 📁 **`tasks/` 目錄必須存在**，不存在時先執行 `mkdir -p tasks/`
- 📝 **需求議題不要含空格**，使用 `-` 或 `_` 連接（例如：`使用者登入-API`）
- 🕐 **時間以執行當下為準**，格式為 `HHmmss`（6 位數）
- 📌 **每次任務寫一份**，不要把多個任務合併到同一個檔案
